# Быстрый старт

Минимальная инструкция, чтобы поднять проект с нуля.

## 1. Что нужно

Только **Docker** с Docker Compose. Больше ничего ставить не надо — ни PHP, ни Node.

Проверить:

```bash
docker --version
docker compose version
```

## 2. Запуск

```bash
make init
```

Эта команда сделает всё сама:

1. создаст `.env` из `.env.example` (имя проекта = имя каталога, UID/GID хоста);
2. поднимет общий reverse-proxy (Traefik), если ещё не запущен;
3. соберёт Docker-образы;
4. **создаст каркасы приложений** в пустых `backend/` (Laravel + Filament) и
   `frontend/` (Next.js) и подключит их к окружению;
5. прогонит миграции и поднимет стек.

> Skeleton поставляется с пустыми `backend/`/`frontend/` — приложения генерируются
> при первом `make init`. Дальнейшая начинка (модули, дизайн-система) подключается
> зависимостью `package-core`.
>
> Первый запуск долгий — собираются образы и каркасы. Дальше — секунды.

## 3. Проверка

URL зависят от имени каталога проекта (`<project>`):

| Что          | Адрес                                  |
| ------------ | -------------------------------------- |
| Сайт (SSR)   | `http://<project>.localhost`           |
| Админка      | `http://api.<project>.localhost/admin` |
| API          | `http://api.<project>.localhost/api`   |

`*.localhost` работает сразу — `/etc/hosts` править не нужно. Несколько проектов
можно держать поднятыми одновременно — у каждого свой хост.

## 4. Каждый день

```bash
make up        # поднять
make down      # остановить
make logs      # смотреть логи
make ps        # статус контейнеров
make help      # все команды
```

## 5. Если что-то не так

- **Порт 80 занят** — его держит общий Traefik (`docker/proxy`). Это нормально: один
  прокси на все проекты. Останавливать сторонний веб-сервер на :80.
- **Сайт не открывается** — проверьте, что прокси поднят (`make proxy-up`) и роутеры
  видны в дашборде http://localhost:8080.
- **Порты БД/Redis** не фиксированы (эфемерные) — узнать: `make ports`.
- **Права на файлы** — проверьте `UID`/`GID` в `.env` (`id -u` / `id -g`), затем
  `make build && make up`.
- **Чистый старт** — `make down`, удалить тома: `docker compose down -v`, потом `make init`.

# Общий reverse-proxy (Traefik)

Один Traefik на всю машину маршрутизирует запросы на все локальные проекты,
поднятые на базе skeleton. Благодаря этому проекты работают **параллельно**:
порт `:80` принадлежит только прокси, отдельные проекты его не публикуют.

## Как это работает

- Traefik слушает `:80` и смотрит в Docker (по лейблам контейнеров).
- Каждый проект помечает свои контейнеры лейблами `traefik.*` (см. `compose.yaml`)
  и подключается к общей внешней сети `proxy`.
- Маршрутизация по хосту:
  - `http://<project>.localhost` → Next.js проекта (SSR);
  - `http://api.<project>.localhost` → Laravel проекта (API + `/admin`).
- `<project>` берётся из `PROJECT_DOMAIN` в `.env` проекта.

> `*.localhost` резолвится в `127.0.0.1` браузерами автоматически — править
> `/etc/hosts` не нужно.

## Запуск

```bash
make proxy-up      # создаёт сеть proxy и поднимает Traefik (один раз на машину)
make proxy-down    # останавливает Traefik
```

Дашборд Traefik (список роутеров/сервисов): http://localhost:8080
